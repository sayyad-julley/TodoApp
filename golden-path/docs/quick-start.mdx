Get started with Golden Path in just a few steps. This guide will walk you through creating your first project using our templates.

## Prerequisites

Before you begin, ensure you have:

- **Node.js** 18+ installed
- **Git** for version control
- **PostgreSQL** 12+ (for templates that use it) or **MongoDB** (for fullstack-todo template)
- **npm** or **yarn** package manager
- (Optional) **Docker** for containerized development

<CodeGroup>
```bash npm
npm --version
```

```bash yarn
yarn --version
```

```bash node
node --version  # Should be >= 18.0.0
```
</CodeGroup>

## Installation

### 1. Clone the Repository

```bash
git clone <repository-url>
cd TodoApp-GoldenPath
```

### 2. Choose Your Method

You have two options to create a new project:

<AccordionGroup>
<Accordion title="Option 1: Using Backstage (Recommended)">
Best for teams using Backstage for software catalog and templates.

#### Start Backstage

```bash
cd todo-backstage
yarn install
yarn tsc
yarn build:backend
yarn start
```

Backstage will be available at `http://localhost:3000`.

#### Create Project

1. Navigate to **"Create Component"** in Backstage
2. Select **"Golden Path Fullstack Todo"** template
3. Fill in project details:
   - **Project Name**: `my-todo-app` (lowercase with hyphens)
   - **Description**: Brief description
   - **Feature Type**: Choose `feature`, `bugfix`, or `refactoring`
   - **Feature Description**: What you want to build
   - **Repository Location**: Your GitHub repo URL

4. Click **"Create"**

The template will:
- âœ… Fetch the base golden path template
- âœ… Generate a Traycer implementation plan
- âœ… Create an implementation guide
- âœ… Publish to your GitHub repository
- âœ… Register in Backstage catalog
</Accordion>

<Accordion title="Option 2: Using the Script">
Best for local development without Backstage.

#### Run the Script

```bash
./create-todo-project.sh -n "my-todo-app" \
  -d "A simple todo application" \
  -t "feature" \
  -f "Create a basic todo CRUD interface"
```

#### Setup Generated Project

```bash
cd ../my-todo-app
npm install
cp backend/env.example backend/.env
# Edit backend/.env with your database credentials
npm run migrate
npm run dev
```
</Accordion>
</AccordionGroup>

## Environment Setup

### Required Environment Variables

Create `backend/.env` file:

```bash
# Database Configuration
DATABASE_URL=postgresql://username:password@localhost:5432/my-todo-app
MONGO_URI=mongodb://localhost:27017/todoapp  # For MongoDB-based templates

# JWT Configuration
JWT_SECRET=your-super-secret-jwt-key-here  # Generate with: openssl rand -base64 32
JWT_EXPIRES_IN=7d

# Server Configuration
PORT=5000
NODE_ENV=development
FRONTEND_URL=http://localhost:3000
```

<Warning>
Never commit `.env` files to version control. They contain sensitive credentials.
</Warning>

### Generate Secure Secrets

```bash
# Generate JWT secret
openssl rand -base64 32

# Generate database password
openssl rand -base64 24
```

## Running Your Application

### Start Development Servers

```bash
# Start both frontend and backend
npm run dev

# Or start separately
npm run dev:frontend  # Runs on http://localhost:5174
npm run dev:backend   # Runs on http://localhost:5000
```

### Verify Installation

1. **Check Backend Health**
   ```bash
   curl http://localhost:5000/health
   ```
   Should return: `{"status":"ok"}`

2. **Open Frontend**
   Navigate to `http://localhost:5174` in your browser

3. **Test Authentication**
   - Register a new user
   - Login with credentials
   - Create your first todo

## Troubleshooting

### Common Issues

<AccordionGroup>
<Accordion title="Port Already in Use">
**Error**: `Port 3000 or 5000 already in use`

**Solution**:
```bash
# Kill process using port 3000
lsof -ti:3000 | xargs kill -9

# Kill process using port 5000
lsof -ti:5000 | xargs kill -9

# Or change ports in configuration
```
</Accordion>

<Accordion title="Database Connection Failed">
**Error**: `Cannot connect to database`

**Solution**:
1. Verify database is running:
   ```bash
   # PostgreSQL
   pg_isready
   
   # MongoDB
   mongosh --eval "db.adminCommand('ping')"
   ```
2. Check credentials in `backend/.env`
3. Ensure database exists:
   ```bash
   # PostgreSQL
   createdb my-todo-app
   
   # MongoDB - database is created automatically
   ```
</Accordion>

<Accordion title="Dependencies Installation Fails">
**Error**: `npm install` fails

**Solution**:
```bash
# Clear cache and reinstall
npm cache clean --force
rm -rf node_modules package-lock.json
npm install
```
</Accordion>
</AccordionGroup>
