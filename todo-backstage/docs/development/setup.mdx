# Development Setup

This guide covers setting up your development environment for working with the Backstage template.

## Prerequisites

- Completed [Installation](/installation)
- Node.js 18+ and Yarn installed
- Git configured
- VS Code (recommended) or your preferred IDE

## Development Workflow

### Local Development

<Steps>
  <Step title="Start Development Server">
    ```bash
    cd todo-backstage
    yarn start
    ```
  </Step>
  <Step title="Open Browser">
    Navigate to http://localhost:3000
  </Step>
  <Step title="Login as Guest">
    Click "Enter" to use guest authentication
  </Step>
</Steps>

## Frontend Development

### Directory Structure

```
packages/app/src/
├── components/        # Reusable React components
├── pages/            # Page components
├── hooks/            # Custom React hooks
├── themes/           # Material-UI themes
├── utilities/        # Helper functions
└── App.tsx           # Main application component
```

### Adding New Pages

1. Create page component in `packages/app/src/pages/`
2. Add route in `packages/app/src/App.tsx`
3. Add navigation item in `packages/app/src/components/Root/`

```typescript
// packages/app/src/pages/MyPage.tsx
import React from 'react';

export const MyPage = () => (
  <div>
    <h1>My Custom Page</h1>
    <p>Welcome to my custom Backstage page!</p>
  </div>
);
```

### Creating Components

```typescript
// packages/app/src/components/MyComponent.tsx
import React from 'react';

interface MyComponentProps {
  title: string;
  description?: string;
}

export const MyComponent: React.FC<MyComponentProps> = ({
  title,
  description
}) => (
  <div>
    <h2>{title}</h2>
    {description && <p>{description}</p>}
  </div>
);
```

import { makeStyles } from '@material-ui/core/styles';

## Backend Development

### Directory Structure

```
packages/backend/src/
├── plugins/          # Custom backend plugins
├── services/         # Core services
├── database/         # Database configurations
├── config/          # Configuration files
└── index.ts         # Backend entry point
```

### Creating Custom Plugins

1. Create plugin directory in `packages/backend/src/plugins/`
2. Implement plugin interface
3. Register plugin in `packages/backend/src/index.ts`

```typescript
// packages/backend/src/plugins/my-plugin/plugin.ts
import { createRouter } from '@backstage/plugin-permission-node';
import { Router } from 'express';

export default async function createPlugin(): Promise<Router> {
  return await createRouter({
    config: {},
  });
}
```

### Custom API Endpoints

```typescript
// packages/backend/src/plugins/my-plugin/router.ts
import { Router } from 'express';
import { errorHandler } from '@backstage/backend-common';

export async function createRouter(): Promise<Router> {
  const router = Router();

  router.get('/health', (_, res) => {
    res.json({ status: 'ok' });
  });

  router.use(errorHandler());
  return router;
}
```

## Template Development

### Creating New Templates

1. Create template in `examples/template/`
2. Define `template.yaml`
3. Add skeleton files
4. Test with Backstage scaffolder

```yaml
# examples/template/my-template/template.yaml
apiVersion: scaffolding.backstage.io/v1beta3
kind: Template
metadata:
  name: my-template
  title: My Custom Template
spec:
  type: service
  parameters:
    - title: Project Information
      required: ['name', 'description']
      properties:
        name:
          title: Name
          type: string
        description:
          title: Description
          type: string
  steps:
    - id: fetch
      action: fetch:template
      input:
        url: ./skeleton
```

## Database Development

### Local Development

For development, the template uses SQLite by default:

```yaml
# app-config.local.yaml
database:
  client: better-sqlite3
  connection: ':memory:'
```

### PostgreSQL Setup

For production-like development:

```yaml
# app-config.local.yaml
database:
  client: pg
  connection:
    host: localhost
    port: 5432
    user: postgres
    password: password
    database: backstage_dev
```

### Migrations

Database migrations are handled automatically by Backstage.

```bash
# Run migrations manually (if needed)
yarn workspace backend start
```

### Common Issues

<AccordionGroup>
<Accordion title="Hot Reloading Not Working">
**Solution**: Check that file watching is enabled and files are being saved properly.
</Accordion>

<Accordion title="Template Not Showing">
**Solution**: Verify template YAML syntax and restart backend.
</Accordion>

<Accordion title="Database Connection Errors">
**Solution**: Check database configuration and ensure database server is running.
</Accordion>

<Accordion title="CORS Errors">

</Accordion>

## Performance Tips

### Frontend Performance

- Use React.memo for component optimization
- Implement proper loading states
- Use code splitting for large components
- Optimize bundle size with tree shaking

### Backend Performance

- Use database connection pooling
- Implement proper caching strategies
- Monitor API response times
- Use appropriate HTTP caching headers
